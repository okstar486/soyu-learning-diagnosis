## 2. 대상 아이 프로파일

### 2.1 기본 정보

| 항목 | 내용 |
|------|------|
| 이름 | 소유 |
| 나이 | 초6 → 중1 (입학까지 약 2개월) |
| 사용 기기 | 갤럭시탭 14.6인치 (아빠 소유, 통제권 아빠) |

### 2.2 확증된 능력 (정상 이상)

| 영역 | 증거 |
|------|------|
| **읽기 능력** | 귀멸의칼날 소설 1,480단어 × 50부작 = **74,000단어** 혼자 독파, 반복 읽기, 암기, 꿈에 나올 정도 몰입 |
| **이해력** | 귀멸 소설 내용 완벽 이해 + 외움 |
| **집중력** | 게임 고도 몰입, 동기 있는 날 학습도 집중 가능 |
| **도전정신** | 게임 어려운 미션 → 계속 도전, "깰 수 있다"는 믿음 |

### 2.3 핵심 문제

| 문제 | 증거 |
|------|------|
| **학습 = 불가능한 성벽** | 게임은 "깰 수 있다" / 공부는 시도 자체 거부 |
| **학습된 무기력** | 모르면 찍기, 빈칸, 거짓말, 대충 마무리 |
| **회피 고착화** | 쓰기 거부, 주관식 회피, 긴 지문 회피 |
| **동기 의존도 극심** | 보상 있으면 몰입 / 없으면 완전 저하 |
| **국어 어휘력 → 전과목 병목** | 수학·영어 선생님 공통 지적 |

### 2.4 쓰기/연필 문제 (중요)

| 항목 | 상태 |
|------|------|
| 연필 잡기 | ❌ 90도 수직에 가깝게 잡음, 교정 실패 |
| 쓰기 속도 | ❌ 느림, 글씨 못 씀, 줄/크기 불일정 |
| 신체 증상 | ❌ 손 아픔, 손목 돌림, 몸 웅크림, 목/허리 통증 |
| 지속력 | ❌ 짧음, 자세 교정 시 집중 저하 |
| 타이핑 | ⭕ 보통 수준, 사용량 증가 필요 |

**→ 설계 원칙: 쓰기 최소화, 타이핑/선택형 중심**

### 2.5 관심사 및 몰입 포인트

| 항목 | 상세 |
|------|------|
| **귀멸의칼날** | 최애: 토키토, 젠이츠, 멧돼지가면 |
| **하루 캐릭터** | 본인이 직접 만든 자아 투영 캐릭터 (보라색 머리, 구미호 가면, 여우의 호흡) |
| **하루 소설** | 50부작 약 74,000단어 완독 + 암기 + 꿈에 나옴 |
| **게임** | 피크민 → 마인크래프트 → 로블록스 (현재 최애 보상) |

### 2.6 학습 환경

| 항목 | 상태 |
|------|------|
| 수학 과외 | 주 2회, 2시간, 기초 확보됨, 국어 이해력이 병목 |
| 영어 과외 | 주 2회, 1.5시간, 주입식 암기, 흥미 낮음 |
| 국어 | 책나무 공부방 (스트레스 적음, 본인 선택, 자기 만족) |
| 숙제 | 엄마 앞에서만, 혼자 두면 딴짓 |

## 3. 핵심 인사이트

### 3.1 근본 진단

```
┌─────────────────────────────────────────────────────────┐
│  "능력 부족"이 아니라 "학습에 대한 자기효능감 붕괴"      │
│                                                         │
│  게임: "나는 깰 수 있다" → 도전 → 성공 → 더 도전        │
│  공부: "나는 못한다" → 회피 → 실패 → 더 회피            │
│                                                         │
│  귀멸소설 74,000단어를 읽은 아이가                       │
│  학습지문을 못 읽는 건 능력 문제가 아님                  │
└─────────────────────────────────────────────────────────┘
```

### 3.2 설계 원칙

| # | 원칙 | 구현 방법 |
|---|------|----------|
| 1 | 쓰기 최소화 | 선택형/타이핑/음성 활용 |
| 2 | 관심사 연결 | 귀멸의칼날 + 하루 캐릭터 전면 활용 |
| 3 | 게이미피케이션 | 레벨업 + 로블록스 보상 연결 가능 구조 |
| 4 | 마이크로 세션 | 15~30분 단위 |
| 5 | 실패 없는 설계 | 난이도 자동 조절, 성공 경험 누적 |
| 6 | 강점 먼저 | 자신감 회복 우선 |

### 3.3 진단 목표

```
1. 원인 파악: 왜 힘든가? (능력 vs 동기 vs 정서 vs 학습스타일)
2. 현재 수준: 어휘력, 읽기 이해 등 객관적 수준
3. 강점 발견: 자신감 재료 확보
4. 학습 동기 평가: 회피 패턴, 동기 유형
5. 맞춤 전략: 부모용 구체적 개입 포인트
```

## 5. 진단 영역

### 5.1 영역 목록 및 우선순위

| 순서 | 영역 | 목적 | 문항 수 |
|------|------|------|--------|
| 1 | 어휘력 | 전과목 병목 확인 | 20~30 |
| 2 | 학습 자기효능감 | 근본 원인 파악 | 15~20 |
| 3 | 학습맥락 읽기 이해 | 지문 이해력 | 10~15 |
| 4 | 학습 동기/정서 | 회피 패턴 분석 | 15~20 |
| 5 | 문법 기초 | 기초 점검 | 15~20 |
| 6 | 강점 발견 | 자신감 재료 | 10~15 |

### 5.2 영역별 상세

#### 5.2.1 어휘력

```yaml
목적: 전과목 학습의 병목인 어휘력 수준 파악
측정 요소:
  - 어휘 폭 (양): 알고 있는 단어 수
  - 어휘 깊이: 다의어, 연어, 용법 이해
  - 어휘 사용: 실제 맥락에서 활용
문제 유형:
  - 동의어/반의어 찾기 (선택형)
  - 빈칸에 적절한 단어 (선택형)
  - 문맥에서 의미 추론 (선택형)
난이도: 초4 ~ 중1
귀멸 테마 적용:
  - 귀멸 캐릭터/상황으로 예문 구성
  - "혈귀의 이름을 외워야 해!" 컨셉
```

#### 5.2.2 학습 자기효능감

```yaml
목적: 학습에 대한 자신감과 믿음 수준 파악
측정 요소:
  - 학습 능력에 대한 자기 인식
  - 실패 후 회복 탄력성
  - 도전 vs 회피 성향
  - 목표 설정 및 달성 기대
문제 유형:
  - 리커트 척도 (5점)
  - 상황 제시 후 반응 선택
  - "나라면 어떻게 할까?" 시나리오
귀멸 테마 적용:
  - 하루 캐릭터의 훈련 상황 활용
  - "하루가 이런 상황이라면?"
```

#### 5.2.3 학습맥락 읽기 이해

```yaml
목적: 학습 상황에서의 지문 이해력 파악
측정 요소:
  - 사실적 이해 (세부정보, 중심내용)
  - 추론적 이해 (생략정보, 의도)
  - 구조 파악 (글의 흐름)
문제 유형:
  - 짧은 지문 + 선택형 문제
  - 중심 내용 찾기
  - 빈칸 추론
난이도: 초5 ~ 중1
귀멸 테마 적용:
  - 귀멸 세계관 배경 지문 (하루 소설 스타일)
  - 익숙한 캐릭터/상황으로 흥미 유발
```

#### 5.2.4 학습 동기/정서

```yaml
목적: 학습 회피 패턴과 동기 유형 분석
측정 요소:
  - 내적 동기 vs 외적 동기
  - 학습 불안 수준
  - 회피 상황 및 트리거
  - 성공/실패 귀인 양식
문제 유형:
  - 상황별 감정/행동 선택
  - "이럴 때 나는..." 시나리오
  - 선호도 순위 매기기
귀멸 테마 적용:
  - 하루의 훈련 과정에서 비유
  - 캐릭터 감정에 공감하며 자기 탐색
```

#### 5.2.5 문법 기초

```yaml
목적: 국어 문법 기초 수준 점검
측정 요소:
  - 품사 구분
  - 문장 성분
  - 맞춤법/띄어쓰기
  - 높임 표현
문제 유형:
  - 선택형 위주
  - 틀린 부분 찾기
  - 올바른 표현 선택
난이도: 초5 ~ 중1
귀멸 테마 적용:
  - 캐릭터 대사 교정
  - "시노부 선생님의 문법 수업"
```

#### 5.2.6 강점 발견

```yaml
목적: 자신감 회복을 위한 강점 재료 확보
측정 요소:
  - 학습 스타일 (시각/청각/체험)
  - 강점 영역 (언어/논리/예술/관계)
  - 성공 경험 회상
  - 흥미 분야
문제 유형:
  - 선호도 선택
  - "내가 잘하는 것" 체크리스트
  - 성공 경험 회상 (선택형)
귀멸 테마 적용:
  - "너의 호흡을 찾아라"
  - 각 호흡 유형 = 강점 유형 매칭
```

| 7 | 여우의 호흡 완성 | 전체 완료 | 졸업/마스터 연출 |

### 6.4 결과 공개 방식

| 대상 | 공개 방식 |

### 6.5 UI/UX 원칙

| 원칙 | 구현 |
|------|------|
| 쓰기 최소화 | 모든 입력은 선택형 또는 타이핑 |
